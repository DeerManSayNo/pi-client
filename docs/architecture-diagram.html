DeerHux 架构图

基于当前项目目录与 AGENTS.md 梳理：前端 React/Next.js UI、API Route、进程内 AgentSession、会话文件读取、SSE 流、配置/索引/并行任务/定时任务等模块的关系。

项目：deerhux
栈：Next.js 16 · React 19 · Tauri 2
运行：npm run dev · port 30141
绿色链路：会话浏览
直接读取 ~/.deerhux/agent/sessions 下 JSONL，不创建 AgentSession。
青色链路：实时对话
POST 命令进入 rpc-manager，创建/复用进程内 AgentSession，并通过 SSE 回推事件。
紫色链路：配置能力
模型、角色、系统提示词、MCP、Memory、Skills 等面板对应独立 API 与 ~/.deerhux/agent 配置文件。
琥珀/玫瑰链路：任务扩展
Scheduler 使用 node-cron；parallel/isolated runs 使用 orchestrator、worker session 与 worktree 执行。
